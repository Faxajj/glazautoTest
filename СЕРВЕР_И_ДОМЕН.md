# Куда лучше поставить дашборд и как получить свою ссылку

Весь проект уже сохранён у тебя в папке `banks-dashboard`. Перед выкладкой на сервер можно собрать архив: запусти **`pack_for_deploy.ps1`** в PowerShell из этой папки — получится `deploy-banks-dashboard.zip` со всеми нужными файлами.

---

## Рекомендация: **VPS + свой домен**

Для этого дашборда (Python, FastAPI, постоянная работа) лучше всего подходит **VPS (виртуальный сервер)** и **свой домен** (например **glazauto.pro**). Тогда у тебя будет одна нормальная ссылка вида:

- **https://glazauto.pro** — без порта, без «server207.hosting.reg.ru», без редиректов.

Почему так:

| Критерий | VPS (свой сервер) | Shared-хостинг (типа Reg.ru) |
|----------|-------------------|------------------------------|
| Своя ссылка (домен без порта) | ✅ Полный контроль: Nginx + SSL | ❌ Часто нельзя нормальный прокси, только редирект или порт в URL |
| Приложение всегда запущено | ✅ systemd — перезапуск при падении и после перезагрузки | ❌ Нужен CRON, процессы могут убивать |
| Python / своё приложение | ✅ Ставишь что нужно | ❌ Ограничения, своя среда не всегда есть |
| Стоимость | От ~200–300 ₽/мес | Дешевле, но для такого дашборда неудобно |

Итог: для «запихнуть дашборд и открывать по своей ссылке» — **оптимально использовать VPS**.

---

## Что делать по шагам

### 1. Есть VPS (например 194.58.95.139)

- Домен **glazauto.pro** привязать к VPS: в DNS сделать A-записи **@** и **www** на IP этого VPS.
- Поставить дашборд по инструкции **`VPS_SETUP_glazauto.md`** (Nginx, Python, venv, systemd, Certbot).
- В итоге дашборд будет доступен по **https://glazauto.pro** — это и есть «своя ссылка».

### 2. VPS ещё нет

Можно взять недорогой VPS у тех же провайдеров, где уже есть домен:

- **Reg.ru** — VPS (если домен там, удобно в одном кабинете).
- **Timeweb**, **Selectel**, **RuVDS** — тоже нормальные варианты для такого проекта.

Достаточно минимального тарифа (1 ядро, 1 ГБ RAM). Домен привязываешь к VPS через A-записи, дальше — как в **`VPS_SETUP_glazauto.md`** (подставить свой IP вместо 194.58.95.139).

### 3. Оставить на shared-хостинге (Reg.ru и т.п.)

Если очень нужно оставить там:

- Работать будет по ссылке с портом, например **https://server207.hosting.reg.ru:8015**, или через редирект с домена на этот адрес.
- Свою «красивую» ссылку **https://glazauto.pro** без порта на таком хостинге без поддержки нормального прокси сделать сложно — нужно уточнять у поддержки хостера.

Для постоянной работы дашборда и одной нормальной ссылки **всё равно лучше VPS**.

## Кратко

- **Сохранить:** всё уже в папке `banks-dashboard`; архив для деплоя — `.\pack_for_deploy.ps1` → `deploy-banks-dashboard.zip`.
- **Куда ставить:** лучше всего **VPS + свой домен** → получится одна ссылка вида **https://glazauto.pro**.
- **Как ставить на VPS:** следовать инструкции **`VPS_SETUP_glazauto.md`** (DNS, Nginx, Python, systemd, Certbot).
