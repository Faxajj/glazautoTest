# Один домен glazauto.pro и работа без твоего ПК

Пошагово: дашборд всегда запущен на сервере и открывается по **https://glazauto.pro** (без порта в адресе).

---

## Часть 1. Чтобы дашборд работал на сервере без твоего компа

### Шаг 1.1. Скрипт автозапуска на сервере

1. Открой **Менеджер файлов** в панели Reg.ru.
2. Зайди в **www** → **glazauto.pro** (корень сайта, там где **app**, **venv**, **.htaccess**).
3. Создай новый файл с именем **START_DASHBOARD_ALWAYS.sh**.
4. Вставь в него такой код (путь замени на свой, если папка сайта другая):

```bash
#!/bin/bash
DIR="/var/www/u3436121/data/www/glazauto.pro"
cd "$DIR" || exit 1
if pgrep -f "uvicorn app.main:app" > /dev/null; then
    exit 0
fi
mkdir -p "$DIR/logs"
nohup ./venv/bin/python -m uvicorn app.main:app --host 127.0.0.1 --port 8015 >> "$DIR/logs/dashboard.log" 2>&1 &
echo "Started at $(date)" >> "$DIR/logs/dashboard.log"
```

5. Сохрани файл.
6. Узнай полный путь к этому файлу (в панели часто показывают при просмотре или в свойствах). Обычно это: **/var/www/u3436121/data/www/glazauto.pro/START_DASHBOARD_ALWAYS.sh**

### Шаг 1.2. Сделать скрипт исполняемым

1. Открой **Shell-клиент** в панели.
2. Выполни (подставь свой путь к папке сайта):

```bash
cd /var/www/u3436121/data/www/glazauto.pro
chmod +x START_DASHBOARD_ALWAYS.sh
```

### Шаг 1.3. Запустить дашборд один раз вручную

В том же Shell выполни:

```bash
./START_DASHBOARD_ALWAYS.sh
```

Проверь: открой в браузере **http://server207.hosting.reg.ru:8015** — должен открыться дашборд. Закрой вкладку Shell — дашборд должен продолжать работать (он уже в фоне).

### Шаг 1.4. Добавить задачу в Планировщик CRON

1. В панели Reg.ru открой **Планировщик CRON** (в меню слева).
2. Нажми **«Создать задачу»** / **«Добавить»**.
3. Заполни:
   - **Команда:**  
     `/var/www/u3436121/data/www/glazauto.pro/START_DASHBOARD_ALWAYS.sh`  
     (путь тот же, что в скрипте выше)
   - **Расписание:** раз в минуту. В полях cron обычно: **минута** = `*`, **час** = `*`, **день** = `*`, **месяц** = `*`, **день недели** = `*`  
     Или выбери готовый вариант вроде «каждую минуту».
4. Сохрани задачу.

После этого дашборд будет автоматически подниматься на сервере после перезагрузки или сбоя, без участия твоего ПК.

## Часть 2. Один домен в адресе — https://glazauto.pro

Чтобы в браузере оставался только **glazauto.pro** (без :8015 и без server207), веб-сервер должен проксировать запросы на твой дашборд.

### Шаг 2.1. Проверить .htaccess в корне сайта

1. В **Менеджере файлов** открой корень сайта **www/glazauto.pro**.
2. Найди файл **.htaccess**. Открой его и проверь, что внутри есть блок с **RewriteRule** и флагом **[P]** (прокси), например:

```apache
<IfModule mod_rewrite.c>
    RewriteEngine On
    RewriteCond %{REQUEST_URI} !^/\.well-known/
    RewriteRule ^(.*)$ http://127.0.0.1:8015/$1 [P,L]
</IfModule>
```

3. Если такого нет — создай или замени содержимое **.htaccess** на это (одна строка RewriteRule с `[P,L]`).
4. Сохрани.

### Шаг 2.2. Убрать редирект с index.html (чтобы работал прокси)

Если в корне сайта лежит **index.html** с редиректом на server207:8015, он будет мешать: при заходе на glazauto.pro откроется эта страница и перенаправит на :8015.

1. В корне **www/glazauto.pro** найди **index.html**.
2. Либо удали его, либо переименуй (например, в **index_redirect_backup.html**), чтобы по умолчанию сработал .htaccess и запрос ушёл на 127.0.0.1:8015.

### Шаг 2.3. Проверка

1. Убедись, что дашборд запущен (по шагу 1.3 или дождись CRON).
2. Открой в браузере **https://glazauto.pro** (лучше в режиме инкогнито или после очистки кэша).

- **Если открылся дашборд и в адресной строке остаётся glazauto.pro** — всё сделано: один домен и работа на сервере без твоего ПК.
- **Если появилась ошибка 500 или «Internal Server Error»** — на shared-хостинге часто запрещают прокси в .htaccess. Тогда остаётся два варианта:
  - Написать в **поддержку Reg.ru**: «Нужно настроить обратный прокси для домена glazauto.pro на 127.0.0.1:8015, чтобы сайт отдавался по https://glazauto.pro».
  - Либо перенести сайт на свой VPS и настроить Nginx по инструкции **VPS_SETUP_glazauto.md** — там один домен glazauto.pro настраивается полностью под твоим контролем.

## Краткий чеклист

| № | Действие |
|---|----------|
| 1 | Создать **START_DASHBOARD_ALWAYS.sh** в корне сайта, chmod +x. |
| 2 | Один раз запустить: `./START_DASHBOARD_ALWAYS.sh` в Shell. |
| 3 | Добавить в CRON задачу на выполнение этого скрипта раз в минуту. |
| 4 | В корне сайта в **.htaccess** включить прокси на 127.0.0.1:8015 (RewriteRule с [P]). |
| 5 | Удалить или переименовать **index.html** с редиректом на :8015. |
| 6 | Проверить **https://glazauto.pro**; при 500 — написать в поддержку или использовать VPS. |

После этого дашборд будет «работать на сервере без твоего компа» и по возможности отображаться по одному домену **glazauto.pro** в адресе.
